Foodie Order System

The code defines a simple order-taking system that prompts users for a main ingredient they wish to order. It fetches meal data based on the user input from an external API and randomly selects a meal from the available options. Each order is stored in an array as an object containing details like the meal name, order number, and status. Users can mark their orders as complete, and the system tracks both the full order history and the last order placed, saving this data in session storage for potential future retrieval. The code allows for recursive prompts, enabling users to place multiple orders or complete existing ones seamlessly.
